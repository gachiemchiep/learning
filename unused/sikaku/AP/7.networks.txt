概要

通信プロトコルの標準化
-------------------

OSI基本参照モデル：7層がある、上位層から下位層までは以下のように
    アプリケーション層
    プレゼンテーション層
    セション層
    トランスポート層
    ネットワーク層
    データリンク層
    物理層

プロトコルとサービス
    プロトコル
    サービス：下の層が上の層に提供する機能

TCP／IP

    モデル：
        OSIモデル       |  TCP/IPモデル       | 実装例
    アプリケーション層    |                    | HTTP, SMTP,SNMP
    プレゼンテーション層  |                    | FTP,DHCP,DNS
    セション層           |  アプリケーション層  | POP,Telnet
    -----------------------------------------------------------
    トランスポート層      |  トランスポート層    | TCP,UDP
    ネットワーク層        |  インタネット層     | IP,ICMPなど
    -----------------------------------------------
    データリンク層        |  ネットワーク、イン  | イーサネット
    物理層               |  タフェース層       |

    TCP/IPの通信
        パケット交換方式：データ＝パケット、上位層から下位相まで順番にパケットにヘッダを追加
    MACアドレス             *
    ネットワーク間の通信      **
        必要な情報：受信のIPアドレス+送信のMACアドレス

ノード間のコネクション確立
        コネクション型通信（通信経路を設定）
        コネクションレス型通信（通信経路を設定しない）

TCP／IPの中核プロトコル
    ネットワーク層のプロトコル：IP
    トランスポート層のプロトコル（確立要求パケット：SYN、確認応答パケット：ACK）
        コネクション型：TCP
        コネクションレス型：UDP

IPアドレス
---------

IPアドレスの実体
    IPアドレス管理：NICが管理、ISPに申請
    IPアドレスの表記

IPアドレスの構成
    IPアドレス＝ネットワークアドレス部 + ホストアドレス部
    IPアドレスクラス：ネットワーク部用のビット数が違う
        クラスA：8ビット、 最初のビットの値 ：0
        クラスB：16ビット、最初のビットの値 ：10
        クラスC：24ビット、最初のビットの値 ：110
    サブネットマスク **
    CIDR
        Classless Inter-Domain Routing
    サブネットマスクとIPアドレス
        IP：             11000000 10101000 00001010 10100110
        192.168.10.166  |                              |    |
        サブネットマスク   11111111 11111111 11111111 11110000
        255.255.255.240 |                              |    |
        1の部：ネットワークアドレス、0の部：ホストアドレス


アドレス変換技術とIPv6
    アドレス変換技術
        多くのノードが一つのIPアドレスだけを供用できる
            プライベートIPアドレス、グローバルIPアドレス
        NAT：Network Address Translation
        IPマスカレード: IPMasquerade
    IPv6: 128bビット + IPsecが必須になる
